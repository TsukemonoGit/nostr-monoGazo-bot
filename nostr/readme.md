# monoGazo-bot

Nostr上で動作する画像ボット「もの画像」。集めた画像をランダムに配信したり、リストへの追加・削除を行う。

## セットアップ

### 必要な環境

- Node.js
- npm

### インストール

```bash
npm install
```

### 環境変数

`.env.sample` を `.env` にコピーして設定する。

| 変数名 | 説明 |
|---|---|
| `NSEC_HEX` | ボットの秘密鍵（hex形式） |
| `PUBHEX_HEX` | ボットの公開鍵（hex形式） |
| `TOKEN` | GitHubアクセストークン |
| `SCRIPTPATH` | スクリプトのルートパス |
| `ORNERS_HEX` | 管理者の公開鍵リスト（JSON配列、hex形式） |
| `POINTUSER_PUB_HEX` | ポイント操作権限ユーザーの公開鍵（hex形式） |
| `GIT_AUTHOR_NAME` | gitコミット用の名前 |
| `GIT_AUTHOR_EMAIL` | gitコミット用のメールアドレス |

### 起動

```bash
node monoGazo-bot.js
```

## コマンド一覧

### 通常投稿（リプライ不要）

| コマンド | 説明 |
|---|---|
| `もの画像` / `mono画像` | ランダムにもの画像を返す |
| `もの画像 {番号}` / `mono画像 {番号}` | 指定番号のもの画像を返す |
| `もの画像 どこ?` | ボットが呼ばれたことを通知 |
| `もの画像サイト どこ?` | もの画像サイトのURLを返す |
| `ないんふぉふぉ` / `ないふぉふぉ` | あるんふぉふぉ画像を返す |
| `あるんふぉふぉ` / `あるふぉふぉ` | アイコンをランダム変更して投稿 |
| `もの、{A}vs{B}して` | vs形式の選択肢からランダムに返す |
| `もの、ランダムNIPして` | NIP-00〜99をランダムに返す |
| `もの、方向?` / `もの、どっち?` | ランダムな方向矢印を返す |
| `ポイント {値} {コメント}` | ポイント記録（権限者のみ） |

### リプライコマンド（ボットへのリプライ）

| コマンド | 説明 |
|---|---|
| `和暦` | 今日の和暦を返す |
| `もの画像 length` / `枚数` / `何枚` | 現在の画像総数を返す |
| `もの画像 {番号}` | 指定番号のもの画像を返す（詳細情報付き） |
| `あるんふぉふぉください` | あるんふぉふぉ画像をリプライで返す |
| `あるんふぉふぉあげて` | あるんふぉふぉ画像を投稿 |
| `{npub}にあるんふぉふぉを送って` | 指定ユーザーにあるんふぉふぉを送る |
| `追加 {nevent/note}` | 画像をリストに追加（管理者のみ） |
| `追加 {nevent/note} {番号}` | 複数画像の場合に番号指定で追加 |
| `削除 {番号}` | 指定番号の画像をリストから削除 |
| `{A}vs{B}して` | vs形式の選択肢からランダムに返す |
| `方向?` / `どっち?` | ランダムな方向矢印を返す |

### `追加` (`add`) コマンドの詳細

管理者がリプライで画像をもの画像リストに追加する。

```
追加 nostr:nevent1...        → イベントから画像を取得して追加
追加 note1...                → 同上
追加 nevent1... 2            → 複数画像の2番目を指定して追加
add nostr:note1...           → 英語コマンドも対応
```

- `nostr:` プレフィックスはあってもなくてもOK
- 対象イベントに画像が1枚の場合 → 自動追加
- 対象イベントに画像が複数の場合 → 番号未指定ならリプライで選択肢を表示し、番号をリプライで返して確定

### `削除` (`delete`) コマンドの詳細

管理者または画像の投稿者が削除可能。

```
削除 42       → index 42 の画像を削除
delete 42    → 同上
```

## データ構造

画像リストは `imageList.json` に保存。追加時に自動で日付順にソートされ、git pushでリポジトリに反映される。

```json
{
  "url": "https://example.com/image.jpg",
  "author": "npub1...",
  "date": "2024/01/01",
  "note": "note1...",
  "memo": ""
}
```
